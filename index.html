路線 B：開店平台 SaaS ＋ 雲端進銷存串接

具體組合 EasyStore 成長版 ＋ CMT 雲端進銷存： 上線快、無平台交易抽成、進貨／成本／多倉／對帳一次到位，且不必養工程師。

首年約 NT$22.7 萬（建置＋12 個月平台與進銷存月費＋金流手續費）、 次年起約 NT$19.7 萬／年。若要「先輕量起步、日後再接進銷存」，可先用路線 A（EasyStore 商務版）首年約 15 萬。

理由：代理單一品牌、SKU 中小量（估上百個）、既有線下庫存需要成本對帳——開店平台四家皆只有前台基本庫存、沒有進貨成本與對帳，故需外接雲端進銷存；全客製（含 ERP）動輒 80 萬起，此規模不符效益。詳見 §4、§5。

為何不選路線 D（全客製）

  • 成本失衡：電商＋ERP 全客製 80 萬–數百萬，中小量 SKU 的 ROI 偏低。
  • 工期過長：2–8 個月起；SaaS 4–8 週即可上線。
  • 維護綁死：長期倚賴原開發商或自聘工程師（月薪 8–12 萬）。
  • 規模不對稱：單品牌業務流程單純，用高成本解決套件即可滿足的需求。

5推薦與導入時程

主推

EasyStore 成長版 ＋ CMT 雲端進銷存（路線 B）。首年約 NT$22.7 萬、次年約 NT$19.7 萬。

三個決定性理由：① EasyStore 不收交易抽成、開放 API，長期成本低於 SHOPLINE／Cyberbiz（免 GMV 抽成）；② CMT 是本次唯一官方同時 API 串接 SHOPLINE／EasyStore／Cyberbiz＋蝦皮的進銷存，未來換前台或多通路並行都不用換後台；③ 一次補上四家開店平台都缺的進貨成本與對帳，且全程 SaaS、不必養工程師。

導入時程（估 5–8 週，約 1.5–2 個月）

  1. 第 1 週 申請與開店：註冊 EasyStore、選成長版、開 14 天試用；同步向綠界／藍新申請金流（送公司登記與負責人證件）。
  2. 第 1–2 週 金流審核＋品牌視覺：金流開通審核約 3–7 個工作天（需預留）；期間完成 LOGO／版型／分類與運費設定。
  3. 第 2–4 週（關鍵路徑） 商品上架：上百個 SKU 的拍照、規格、文案、定價——這是最耗時的一段，視人力 2–4 週。
  4. 第 3–4 週 串接 CMT 進銷存：匯入既有庫存／成本／供應商，設定進貨與對帳流程、與 EasyStore 訂單自動拋轉。
  5. 第 4–5 週 試營運：金流／物流實測下單、庫存扣帳與對帳驗證、少量真實訂單試跑。
  6. 第 5–6 週 正式上線：搭配蝦皮既有客群導流、經銷 DM 放官網 QR。

關鍵路徑＝商品上架人力。若能先備妥商品照與規格表，整體可壓到 5 週內。

6風險與注意事項

  • 平台綁定（vendor lock-in）：SaaS 的商品／會員資料匯出格式受限，日後換平台需搬資料。因應：選有開放 API 與匯出的平台（EasyStore 具備），且把進銷存（CMT）獨立於前台，換前台不動後台。
  • 資料轉移：既有 Excel 庫存／客戶／成本資料匯入需先清洗欄位，對齊 CMT 匯入格式；建議上線前做一次完整對帳校驗。
  • 金流審核時間：綠界／藍新開通需送件審核，約 3–7 個工作天甚至更久，務必提前於專案第 1 週申請，避免卡上線。
  • GMV 抽成陷阱：SHOPLINE、Cyberbiz 除年費外另抽 GMV（1%–5%），營收越高抽越多；試算總成本務必含這塊（本報告主推的 EasyStore／WACA 不抽成）。
  • 海外主機續約跳價（路線 C）：SiteGround 等首年促銷價與續約價可差 5 倍，若走自架須以續約價估長期成本。
  • 價格時效：平台方案頻繁改版與促銷，本報告數字為 2026-07-07 查證之官方現行價；正式決策前請以官網當下頁面或業務書面報價為準。
